GitHub アップロード手順書

あなたのパソコンの中にあるシステムを GitHub に上げて、
仲間みんなで一緒に開発できるようにするまでの完全ガイドです。

📌 このマニュアルについて

「自分のパソコンの中だけで作ったシステム」を、GitHub というネット上の保管場所にアップして、他のメンバーも開発に参加できる状態にします。専門用語はそのつど説明するので、Git をさわったことがなくても大丈夫です。

👤 全員プログラミング初心者 🤖 Claude Code / Codex で開発 👥 2〜3人のチーム
💡

うれしいお知らせこのマニュアルのコマンド入力は、ほとんど Claude Code や Codex に「やって」とお願いするだけでも進められます。ただし「何が起きているか」を理解しておくとトラブルに強くなるので、各ステップに説明をつけています。

この手順書のステップ

  1. GitHub のアカウントを作る
  2. Git をパソコンに入れる
  3. アップする前に「秘密ファイル」を守る設定
  4. GitHub に空っぽの保管場所を作る
  5. システムを GitHub にアップする
  6. 仲間を招待する
  7. 仲間が自分のパソコンに取り込む
  8. 用語集 & よくあるトラブル
🧭

誰がどれをやる？
ステップ 1〜5 は「システムを作った本人（アップロードする人）」が行います。
ステップ 6 も同じ人。7 は「あとから参加する仲間」全員が行います。

1

GitHub のアカウントを作る

GitHub（ギットハブ）は、プログラムをネット上で保管・共有できる無料のサービスです。まずは全員がアカウントを持ちましょう。

手順

  • https://github.com を開く
  • 「Sign up（サインアップ）」をクリック
  • メールアドレス・パスワード・ユーザー名を入力して登録
  • 届いたメールの確認コードを入力して完了
📝

決めたユーザー名はメモしておくあとで仲間を招待するときに、お互いの GitHub ユーザー名が必要になります。チームの全員ぶん共有しておきましょう。

💰

料金は無料でOK個人利用なら、非公開（Private）の保管場所も無料で作れます。最初から課金は不要です。

2

Git をパソコンに入れる

Git（ギット）は、ファイルの変更履歴を記録する道具です。GitHub に上げるにはこの Git が必要です。

Mac の場合

多くの Mac には最初から入っています。確認するには「ターミナル」アプリを開いて、次を入力します。

ターミナルgit --version

git version 2.xx.x のように表示されればOKです。「インストールしますか？」と聞かれたら、案内に従って入れてください。

Windows の場合

  • https://git-scm.com/download/win を開く
  • ダウンロードされたファイルを実行
  • 設定は基本そのまま「Next」を押し続けてOK

🤖 Claude Code / Codex に頼むなら

ターミナルの操作が不安なら、AIにこう聞けば確認・案内してくれます。

「Gitが入っているか確認したいです。確認コマンドを教えて、入っていなければインストール方法も教えてください。」
3

アップする前に「秘密ファイル」を守る設定

⚠️ ここが一番大事なステップです。パスワードやAPIキーなどの秘密の情報を、うっかりネットに公開しないための準備をします。

🚨

なぜ必要？GitHub に上げたファイルは、設定によっては世界中から見えてしまいます。.env などに書かれたパスワードやAPIキーが漏れると、悪用されて高額請求などの被害につながります。先に「上げないファイル」を指定しておきましょう。

やること：「.gitignore」というファイルを作る

システムのフォルダの中に .gitignore という名前のファイルを作り、「アップしたくないもの」を書いておきます。これに書いたファイルは GitHub に上がりません。

.gitignore の中身（例）# パスワードやAPIキーなどの秘密情報 .env .env.local # 自動で作られる重いフォルダ（上げる必要なし） node_modules/ __pycache__/ dist/ build/ # パソコンが勝手に作るゴミファイル .DS_Store *.log

🤖 これこそAIにお任せ

自分のシステムに合った .gitignore は、AIが一発で作ってくれます。

「このプロジェクトをGitHubに上げます。秘密情報や不要ファイルを除外する .gitignore を、このフォルダの中身に合わせて作ってください。」
🔑

秘密情報の共有方法.env は GitHub に上げないので、仲間には別の安全な方法（パスワード管理アプリ、暗号化メモ等）で渡します。チャットやメールに直書きするのは避けましょう。

4

GitHub に空っぽの保管場所を作る

アップ先となる「リポジトリ（repository＝保管場所）」を GitHub 上に用意します。

手順

  • GitHub にログインし、右上の 「+」→「New repository」 をクリック
  • Repository name：プロジェクト名を入力（例：legal-tool）
  • Private を選ぶ（← 仲間だけが見られる非公開設定。最初はこれ推奨）
  • 下の「Add a README」などのチェックは全部オフのままでOK
  • 「Create repository」 をクリック
🔒

Public と Private の違い
Private（非公開）＝招待した人だけ見られる。仕事のシステムはこちら。
Public（公開）＝世界中の誰でも見られる。今回は選ばないでください。

作成後の画面に表示される https://github.com/〇〇/△△.git というURLをコピーしておきます。次のステップで使います。

5

システムを GitHub にアップする

いよいよ、自分のパソコンのシステムをネット上の保管場所に送ります。

ターミナルでシステムのフォルダに移動

ターミナル# 「cd 」と打ったあと、フォルダをドラッグ＆ドロップすると楽です cd システムのフォルダの場所

順番にコマンドを実行

ターミナル# ① このフォルダをGitで管理し始める git init # ② 今のファイルを「記録する対象」に全部入れる git add . # ③ 「初回コミット」という名札をつけて記録する git commit -m "初回コミット：既存システムの取り込み" # ④ アップ先のURLを登録する（ステップ4でコピーしたURL） git remote add origin https://github.com/〇〇/△△.git # ⑤ メインの置き場の名前をmainに統一 git branch -M main # ⑥ いよいよGitHubへ送信！ git push -u origin main
🎉

完了！GitHub のリポジトリのページを再読み込みすると、自分のファイルが一覧に並んでいるはずです。これでネット上に保管できました。

🤖 ぜんぶAIに任せてもOK

コマンドを1つずつ打つのが不安なら、Claude Code / Codex にこう頼めば、確認しながら進めてくれます。

「このフォルダを、GitHubのリポジトリ（URL：https://github.com/〇〇/△△.git）に初めてアップしたいです。git init から push まで、順番に実行してください。途中で確認が必要なところは教えてください。」
🔐

パスワードを聞かれたらpush のときにログインを求められます。GitHub では通常パスワードではなく「パーソナルアクセストークン」が必要です。求められたら慌てず、AIに「push でトークンを求められました。作り方を教えて」と聞けば案内してくれます。

6

仲間を招待する

これで他のメンバーも、このシステムを見たり編集したりできるようになります。

手順

  • リポジトリのページで 「Settings（設定）」 タブをクリック
  • 左メニューの 「Collaborators（共同作業者）」 をクリック
  • 「Add people」 をクリック
  • 仲間の GitHubユーザー名（ステップ1でメモしたもの）を入力して招待
  • 招待された人にメールが届くので、本人が「Accept（承認）」する
👥

2〜3人ならこれで十分もっと人数が増えたり、会社として管理したくなったら「Organization（組織）」を作る方法もありますが、今の規模ならこのまま個人リポジトリへの招待でOKです。

7

仲間が自分のパソコンに取り込む

👉 ここは「あとから参加するメンバー」全員がやる作業です。GitHub にあるシステムを、自分のパソコンにコピー（クローン）します。

手順

ターミナル# システムを置きたい場所に移動してから実行 git clone https://github.com/〇〇/△△.git # できたフォルダの中に入る cd △△

これで、GitHub にあるシステム一式が手元にそろいました。あとは Claude Code / Codex をこのフォルダで開けば、すぐ開発を始められます。

🔑

.env を忘れずに秘密情報の .env は GitHub に上げていないので、クローンしただけでは入っていません。システムを作った本人から安全な方法で受け取り、フォルダの中に置いてください。これがないとシステムが動かないことがあります。

🤖 開発中の基本サイクル（おまけ）

共同開発で大事なのは「いきなり本体をいじらない」こと。Claude Code / Codex に下記をお願いするのが基本リズムです。

「作業を始める前に、最新の状態を取り込みたい（git pull）」

作業が終わったら →
「変更内容を保存して、GitHubに上げてください（add → commit → push）」
⚠️

同じファイルを同時に編集しない2人が同じファイルを別々に変更すると、あとで「どっちを残す？」という衝突（コンフリクト）が起きます。慣れるまでは「今これ触ってるよ」と声をかけ合うと安全です。

8

用語集 & よくあるトラブル

知っておくと安心な用語

リポジトリ（repository / repo）
システム一式を入れておく「保管場所」。GitHub上に1プロジェクト＝1個作ります。
クローン（clone）
GitHubにある保管場所を、自分のパソコンに丸ごとコピーすること。
コミット（commit）
変更内容に「ここまでやった」という名札をつけて記録すること。セーブポイントのイメージ。
プッシュ（push）
手元の記録（コミット）を GitHub に送ること。「アップロード」に近い。
プル（pull）
GitHub上の最新版を手元に取り込むこと。push の逆。
ブランチ（branch）
本体（main）とは別の「作業用の枝」。慣れてきたら、ここで作業して安全に合流させます。

よくあるトラブルと対処

こんなときどうする
push でログイン/トークンを求められたAIに「トークンの作り方を教えて」と聞く。GitHubの Settings → Developer settings から作成。
秘密情報を間違えて上げてしまったすぐにそのパスワード/キーを無効化して作り直す。消すだけでは履歴に残るので要注意。AIに相談を。
「コンフリクト（衝突）」と出たあわてず作業を止めて、AIに「コンフリクトが出ました。解決を手伝って」と頼む。
どのコマンドか分からなくなったClaude Code / Codex に「今やりたいこと」を日本語で伝えれば、正しいコマンドを出してくれる。
🤝

最後にいちばん大切なこと全員初心者でも、Claude Code / Codex が隣にいれば大丈夫です。「やりたいことを日本語で伝える」だけで、コマンドも手順も教えてくれます。困ったら、まず素直に聞いてみましょう。
